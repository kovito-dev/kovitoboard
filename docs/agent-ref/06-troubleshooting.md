# 06. トラブルシュート

**対象 KB バージョン:** v0.1.0
**最終更新:** 2026-04-18

> 📖 **この章を読むタイミング:** INDEX.md の「🚨 トラブル対応」または他章の「〇〇が表示されない／動かない」から誘導された場合。

## §1 起動しない

### 1.1 症状

- KB のコマンドを実行したが画面が開かない
- ブラウザで `localhost:5173` を開いても真っ白・エラー
- ターミナルにエラーが出て止まる

### 1.2 対処手順

1. **Node.js のバージョンを確認:** v0.1.0 は Node.js 20 以降を前提とします。`node -v` で確認してください
2. **依存が入っているか確認:** `npm install` を実行していないと起動できません
3. **ポート衝突を確認:** `5173` / `3001` が他プロセスで使われていないか
4. **過去のプロセスが残っていないか:** `tmux list-sessions` で古い KB セッションが残っていたら `tmux kill-server` してから再起動
5. **ログを確認:** ターミナル出力の最後のエラーメッセージをユーザーに伝えて一緒に判断

### 1.3 どうしても直らないとき

KB リポジトリ内の `README.md` のトラブルシュート節、または GitHub Issues を確認してください。

## §2 エージェントが表示されない

### 2.1 症状

- エージェント一覧画面にカードが出ない
- 追加したはずのエージェントが出てこない

### 2.2 原因候補と対処

1. **ファイル名と `name` の不一致:** `.claude/agents/` 配下のファイル名（拡張子を除く）と frontmatter の `name` が一致している必要があります
2. **frontmatter が壊れている:** YAML の構文エラーがあるとスキップされます。`---` で挟まれた部分を確認
3. **KB を再読込していない:** ファイル直編集の場合は KB の再起動が必要です
4. **画像が出ないだけで実在する:** アバターはフォールバック生成されるので「画像が出ない = 消えた」ではありません。カードのタイトル領域を確認してください

詳細は [`02-agents.md`](./02-agents.md) を参照してください。

### 2.3 画像が出ない場合

- フォールバック生成は常に動きます（`name` からハッシュ生成したアバター）
- もしフォールバックも出ていない場合、エージェント自体が認識されていません。§2.2 を確認

## §3 セッションが進まない

### 3.1 症状

- メッセージを送ったが応答が返ってこない
- 「応答中...」の表示が消えない

### 3.2 対処手順

1. **Esc キーで割り込み:** エージェントが長考しているだけの可能性があります。一度止めてから短い質問を投げて応答が返るか確認
2. **tmux ペインを確認:** `tmux list-panes` で対象ペインが生きているか
3. **Claude Code のログ確認:** セッションのログに API エラー等が出ていないか（`.claude/logs/` 配下）
4. **セッションを終了して再作成:** 最も確実な復旧策。ただし現セッションのコンテキストは失われます

### 3.3 断続的に固まる場合

- ネットワーク接続を確認（Claude API へのアクセスがタイムアウトしている可能性）
- 短時間に大量のメッセージを送っていないか（レート制限）

## §4 レシピが読み込めない

### 4.1 症状

- `.zip` や `.yaml` をドラッグ＆ドロップしても何も起きない
- URL 指定で「取得に失敗しました」と出る
- 確認画面でエラーメッセージが出てインストール前に止まる

### 4.2 原因候補と対処

1. **ファイル形式が非対応:** v0.1.0 は `.zip` / `.yaml` のみ対応
2. **静的検査で弾かれた:** 禁止された成果物タイプ・scope 範囲外のパス等。エラーメッセージを確認
3. **URL 取得失敗:** ネットワーク接続・URL の HTTPS 対応を確認
4. **起動時スキャナの取りこぼし:** `.kovitoboard/incoming/` 配下にファイルがあるか確認。KB 再起動で再スキャンが走ります

### 4.3 インストール後に動かない場合

- **scope 不足:** `ScopeViolation` エラーが監査ログ（`app/data/{recipe-id}/_audit.log`）に出ている可能性
- **handler 引数エラー:** パスが scope の範囲外だと handler が拒否されます

詳細は [`04-recipes.md`](./04-recipes.md) §8 を参照してください。

## §5 ファイルアクセスエラー

### 5.1 症状

- レシピ実行時に「ファイルが見つかりません」「アクセスが拒否されました」
- `app/api/*.ts` 内で `readFileSync` がエラー

### 5.2 原因候補と対処

1. **パスの基準がずれている:** KB のプロセスは KB リポジトリルートを cwd として動きます。相対パスはルートから解釈してください
2. **scope 範囲外の指定（レシピ）:** レシピから触れるのは scope で宣言した範囲のみ。`.env` や `.git/` は scope に関わらず常に拒否
3. **OS の権限不足:** ユーザーがファイルシステム的に読めないパスは KB も読めません
4. **Windows の path separator:** バックスラッシュとスラッシュの混在に注意（`path.join` を使えば解決）

### 5.3 どこを触れるかの整理

| 区分 | 触れる範囲 |
|---|---|
| レシピ（`project-read`） | プロジェクトルート配下（`.env*`・`.git/`・`node_modules/`・`.claude/credentials*` を除く） |
| レシピ（`own-data`） | `app/data/{recipe-id}/` |
| `app/api/` | Node.js 全域（OS 権限が許す限り） |

## §6 Claude Code が応答しない

### 6.1 症状

- KB 経由で Claude Code を呼ぼうとしても反応がない
- タイムアウトエラーが出続ける

### 6.2 原因候補と対処

1. **Claude Code CLI の認証切れ:** ターミナルで `claude` を直接実行してログイン状態を確認
2. **CLAUDE.md の構文エラー:** KB 起動時に読み込む `.claude/CLAUDE.md` や `CLAUDE.md` に構文エラーがあると止まることがあります
3. **サブエージェント定義の frontmatter 不正:** §2.2 と同じ。YAML が壊れていると対象エージェントが呼べません
4. **Claude API 側の障害:** Anthropic status ページを確認

### 6.3 プロセス系のリカバリ

- `claude` プロセスが残っている場合: `ps aux | grep claude` → 該当 PID を `kill`
- それでも直らない場合は OS 再起動を検討

## §7 trust prompt（信頼確認）が繰り返し出る

### 7.1 症状

- Claude Code が新しいファイル・スキル・ディレクトリを使うたびに信頼確認ダイアログが出る
- 一度承認したはずなのにまた出る

### 7.2 原因と仕組み

Claude Code は、未知のディレクトリ（`.claude/` 配下含む）の実行前に信頼確認を出します。これはセキュリティ機能で、KB が勝手に抑制できるものではありません。

### 7.3 対処

1. **ディレクトリ単位で承認する:** 一度の承認で当該ディレクトリ配下が一括信頼されます
2. **繰り返し出る理由が不明なとき:** Claude Code の trust データベースが壊れている可能性。`.claude/` 配下の trust ファイルを確認（直編集は慎重に）
3. **新しいレシピをインストールした直後:** レシピが追加したファイル群に trust prompt が出るのは正常。承認してください

### 7.4 KB の立ち位置

KB はユーザーに代わって trust を勝手に承認しません。**ユーザー自身が判断する** ことが仕様です。エージェント（コビー含む）も、ユーザーに承認を強く促さず、必要性を説明してから判断を仰いでください。

---

## §8 それでも解決しない場合

- 症状を整理して「Kovito 開発者」エージェントに相談してください
- ログファイル（`.claude/logs/`・`.kovitoboard/logs/`）の直近エントリがあると診断が進みます
- 再現手順を書き出して GitHub Issues に投稿することも検討してください
