## §1 セッションとは

セッションは **1 エージェントとの 1 回の対話単位** です。同じエージェントに対して複数の過去セッションを持つことができ、個々は独立したコンテキストを持ちます。

### 1.1 セッションの実体

- Claude Code のサブプロセスを 1 つ起動し、指定エージェントのシステムプロンプトをロード
- tmux のペインで動作するため、KB を閉じてもセッション自体は継続可能
- セッション ID は内部で UUID を割り振り、UI では「2026-04-18 10:30 〜」のような表記

### 1.2 ライフサイクル

```
作成 → 活動中 → 中断（ユーザー離席） → 再開 → 終了（アーカイブ）
```

- 中断しても状態は保持されます（tmux 側が生きている限り）
- 終了するとアーカイブされ、履歴として参照のみ可能になります

## §2 セッションの開始

### 2.1 エージェント一覧から開始（推奨）

1. エージェントカードをクリック → 詳細画面に遷移
2. 「新しいセッションを始める」ボタンを押す
3. セッション画面に切り替わり、入力欄が表示される

### 2.2 エージェント一覧（ホーム画面）から

エージェント一覧の「最近のエージェント」から素早く起動できます。

### 2.3 最初のメッセージ

最初のメッセージで意図を明確に伝えると、以降の応答精度が上がります。例:

- ❌「何かやって」
- ⭕「今日の TODO を整理してから、優先順位を 3 つ提案してください」

## §3 メッセージ送信と割り込み

### 3.1 送信

入力欄に文章を入れて Enter（または送信ボタン）で送信します。Shift + Enter で改行できます。

### 3.2 エージェントの応答中に割り込む

エージェントが長い応答を出している最中に「やっぱり違う」と思ったら、**Esc キー**（または画面の「停止」ボタン）で割り込みが可能です。

### 3.3 添付ファイル

画像・テキストファイルを入力欄にドラッグ＆ドロップで添付できます。v0.1.0 の対応形式:

- 画像: `png` / `jpg` / `webp`
- テキスト: `md` / `txt` / `json` / `yaml`
- その他: パスを貼り付けての参照（エージェントが Read する）

## §4 過去セッションの参照

### 4.1 エージェント別の履歴

エージェント詳細画面の「過去のセッション」タブから、そのエージェントとの全セッションを時系列で見られます。

### 4.2 横断検索

エージェント一覧画面の検索ボックスから、全エージェントの全セッションを横断検索できます（v0.1.0 ではキーワード一致のみ）。

### 4.3 履歴からの再開

過去セッションを開き、「このセッションを再開する」ボタンで、同じコンテキストを引き継いで続きを話せます。**ただし元のセッションのコンテキストが長いと Claude 側で圧縮が働きます。**

---

## §5 セッションの終了・中断

### 5.1 中断（離席する場合）

- 画面右上の「中断」ボタン、または単に KB を閉じる
- tmux のペインは生きたまま保持され、次回再開時に続きから使える

### 5.2 終了（もう使わない場合）

- 画面右上の「終了」ボタン
- tmux のペインがクリーンアップされ、履歴としてアーカイブされる
- 終了したセッションは「再開」できず、新しいセッションを作る必要がある

### 5.3 自動アーカイブ

- 24 時間以上操作のないセッションは、次回起動時に自動でアーカイブ候補として提示されます
- 自動削除はされません（ユーザー判断）

## §6 tmux の基本（混乱したとき用）

KB は内部で tmux を使ってセッションを管理しています。通常は意識する必要がありませんが、以下のケースで tmux の知識が役立ちます:

### 6.1 KB の外から tmux セッションが見えてしまう

ターミナルで `tmux list-sessions` を打つと KB 管理下のセッションが出てきます。これは正常です。**KB 外から `tmux kill-session` しないでください**（KB の UI 表示と状態が乖離します）。

### 6.2 プロセスが残っている気がする

- KB を正しく終了すれば tmux も一緒に止まります
- 残っている場合は、KB 再起動 → 「セッション一覧」から「全て終了」で一括クリーンアップできます

### 6.3 どうしても壊れた場合

- `tmux kill-server` で全セッションを強制終了（進行中の作業は失われます）
- その後 KB を再起動してください

詳しくは [`06-troubleshooting.md`](./06-troubleshooting.md) §3 を参照してください。
