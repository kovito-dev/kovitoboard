## §1 KovitoBoard とは

KovitoBoard は、Claude Code をエンジン、`.claude/` を土台として動作する **自己拡張型 AI エージェントチーム基盤** です。ユーザーは対話を通じて、自分専用の業務システム（秘書・開発者・リサーチャー等のエージェント集団）を育てていきます。

### 1.1 3 行での説明

- **エージェントと話す。** 画面左のエージェント一覧から相手を選び、セッションを開いて依頼します。
- **レシピを増やす。** よく使う一連の処理はレシピとして配布・受け取りが可能です。
- **アプリを作る。** 独自の UI・API が欲しくなったら `app/` 配下に自分で追加できます。

### 1.2 KB が価値を出す前提

- 手元マシンで動作するローカルアプリケーションであり、クラウド SaaS ではありません。
- Claude Code を前提にしており、マルチ LLM 対応はしません。
- ユーザーは非エンジニア〜ライトエンジニア層を想定しています。対話で完結する体験を重視します。

## §2 基本の画面構成

KB の画面は大きく 3 つのエリアに分かれています。

### 2.1 エージェント一覧（起動直後のホーム）

起動直後に表示される画面です。登録済みのエージェントがカード形式で並び、画像・名前・説明を一覧できます。ここから会話を始めたり、エージェントの詳細を確認できます。迷ったらここに戻ってきてください。詳細は [`02-agents.md`](./02-agents.md) を参照してください。

### 2.2 セッション画面

エージェントを選んで「話しかける」と表示される画面です。メッセージ入力欄・履歴・右側の補助パネル（使用中のレシピ等）から構成されます。詳細は [`03-sessions.md`](./03-sessions.md) を参照してください。

### 2.3 アプリ画面（3 タブ構成）

機能の追加・管理を行う画面で、3 つのタブに分かれています:

- **Apps（アプリ）:** インストール済み・有効なアプリの一覧。ドラッグ＆ドロップでサイドバーの並び順を変更でき、その順序は永続化されます。
- **Sample apps（サンプルアプリ）:** KB 同梱のサンプルレシピ（Document Viewer / TODO）を**有効化／無効化**する場所。無効化は非破壊で、再有効化すると以前のデータを引き継げます。
- **Recipes（レシピ）:** 外部レシピの一覧。**外部レシピのインストールは v0.2.x では無効**で、このタブには「v0.3.0 で KovitoHub と連動して提供予定」の案内が表示されます。

詳細は [`04-recipes.md`](./04-recipes.md)（レシピ）/ [`05-apps.md`](./05-apps.md)（独自アプリ・無効化／削除）を参照してください。

## §3 主要な用語

以下は KB を使ううえで最低限押さえるべき用語です。章をまたいで頻出します。

| 用語 | 一言説明 | 詳細章 |
|---|---|---|
| エージェント | 対話相手となる AI。秘書・コビー等の人格を持つ | [`02-agents.md`](./02-agents.md) |
| セッション | 1 エージェントとの 1 回の対話単位 | [`03-sessions.md`](./03-sessions.md) |
| レシピ | 機能の塊を配布・受け取り可能にしたもの（サンドボックス内で動作） | [`04-recipes.md`](./04-recipes.md) |
| アプリ（app/） | ユーザー独自の UI・API を配置する場所（フル権限、自己責任） | [`05-apps.md`](./05-apps.md) |
| スキル | Claude Code のスキル機能。KB からは読み取り中心で参照 | [`07-advanced.md`](./07-advanced.md) |
| trust prompt | Claude Code が新しいファイル・スキルを使う際に出す信頼確認 | [`06-troubleshooting.md`](./06-troubleshooting.md) §7 |

### 3.1 エージェント vs セッション

エージェントが「相手（人物）」、セッションが「その人との会話 1 回分」です。同じエージェントに対して複数の過去セッションを持つことができます。

### 3.2 レシピ vs アプリ

両者ともユーザーに機能を追加する点は共通ですが、権限境界が明確に異なります:

- **レシピ:** 宣言的な handler でサンドボックス内から KB の機能を呼び出す。scope 承認が必要。
- **アプリ:** `app/` 配下にユーザーが書いた TypeScript を KB が直接実行する。フル権限。

詳細は [`05-apps.md`](./05-apps.md) §5 を参照してください。

## §4 マスタ情報と設定ファイル

### 4.1 マスタ情報（`.kovitoboard/setting.json`）

KB のプロジェクトルート直下の `.kovitoboard/setting.json` が、ユーザー名・プロジェクト名・既定のエージェント等を保持します。

#### 主なフィールド

- `userName`: ユーザーの呼称（エージェントから呼びかける名前）
- `projectName`: プロジェクトの表示名
- `defaultAgentId`: 起動時に既定で選択されるエージェント ID
- `locale`: `ja` または `en`（UI は i18n 対応済み。OSS 既定のフォールバックは `en`）

#### 変更方法

1. 画面右上の設定アイコン → 「マスタ情報」から変更する（推奨）
2. または `.kovitoboard/setting.json` を直接編集する（JSON 構文を壊さないよう注意）

UI からの変更はバリデーション済みです。直接編集した場合は KB を再起動してください。

### 4.2 エージェント定義ファイル（`.claude/agents/*.md`）

各エージェントは `.claude/agents/{agent-id}.md` というファイルで定義されます。Claude Code の規約に従い、frontmatter にメタ情報、本文にシステムプロンプトを書きます。詳細は [`02-agents.md`](./02-agents.md) を参照してください。

### 4.3 画像・添付ファイル

エージェント画像等のバイナリは `.kovitoboard/assets/` 配下で管理します。直接触る必要はありません（UI からアップロードしてください）。

### 4.4 プロジェクトルートの `CLAUDE.md` への KB ガイダンス注入

KB はオンボーディング完了時に、**プロジェクトルート直下の `CLAUDE.md`**（`.claude/CLAUDE.md` ではなく `<project>/CLAUDE.md`）へ、ごく短いガイダンスブロックを **一度だけ** 自動で追記します。注入されるのは次のマーカー付きブロックだけです:

```markdown
<!-- KB:GUIDANCE_START -->
This project uses KovitoBoard (KB) at `./kovitoboard/`. For any
KovitoBoard-related task, start with
`kovitoboard/docs/agent-ref/INDEX.md`.
<!-- KB:GUIDANCE_END -->
```

この目的は、プロジェクト側で動く Claude Code エージェントが「KB 関連の依頼を受けたら、まず agent-ref（このリファレンス）の `INDEX.md` を見る」と気づけるようにすることです。

挙動の要点:

- **冪等（idempotent）:** マーカー対が既に存在すれば再注入しません。ユーザーが既存の `CLAUDE.md` に書いた内容は書き換えず、ブロックを末尾に追記するだけです（`CLAUDE.md` が無ければ新規作成）。
- **ベストエフォート:** ファイルが極端に大きい・マーカーが壊れている・書き込み不可などの場合は注入をスキップし、オンボーディング自体は止めません。
- **オプトアウト可:** オンボーディングウィザードでこの注入を無効化できます。
- **唯一の自動書き込み例外:** このブロックの追記は、KB がプロジェクトルートの `CLAUDE.md` に対して行う **唯一の自動編集** です（§5 の「KB は `.claude/CLAUDE.md` を読むのみ」方針とは対象ファイルが異なります — 注入先は `<project>/CLAUDE.md`、§5 が触らないと約束するのは `.claude/CLAUDE.md`）。

エージェントへの示唆: ユーザーの `CLAUDE.md` 末尾に `KB:GUIDANCE_START` / `KB:GUIDANCE_END` で囲まれたブロックを見つけても、それは KB が自動で置いたものです。手で消しても次のオンボーディングで再注入されうるため、恒久的に外したい場合はウィザードのオプトアウトを案内してください。

## §5 設計哲学（勝手に触らない領域）

KB はユーザーの `.claude/` 配下を **ユーザー自身の領域** として扱います。以下は KB が自動で触らない方針です:

- `.claude/CLAUDE.md`: ユーザーのグローバル指示。KB からは読むのみ（注: プロジェクトルートの `<project>/CLAUDE.md` への一度きりのガイダンス注入は別物です。§4.4 参照）。
- `.claude/settings.json`: Claude Code のフック・権限設定。ユーザー同意なしに編集しない。
- `.claude/agents/*.md`: エージェント定義は UI 操作経由で編集。外から勝手に書き換えない。

エージェント（コビー含む）も、この哲学を守ってください。ユーザーから明示的に依頼されても、`.claude/` 配下の書き換えは必ず差分を提示してから行ってください。
