## §1 エージェントとは

エージェントは、KB 内でユーザーと対話する **AI の人格単位** です。実体は `.claude/agents/{agent-id}.md` というファイルで、Claude Code のサブエージェント定義と同じ形式を使います。

### 1.1 ファイル形式

```markdown
---
name: kovito-concierge
displayName: コビー
description: Kovito のコンシェルジュ。KB の使い方全般を案内します。
model: sonnet
---

あなたは Kovito のコンシェルジュ「コビー」です。
（ここにシステムプロンプトを書く）
```

- `name`: エージェント ID。ファイル名と一致させる（`kovito-concierge.md`）。半角英数字とハイフンのみ。
- `displayName`: UI で表示される名前。日本語可（「コビー」「秘書」等）。
- `description`: エージェント一覧カードに表示される短い説明。
- `model`: 使用する Claude モデル。既定は `sonnet`。
- 本文: システムプロンプト。人格・口調・守るべきルール等を記述。

### 1.2 「名前」と「表示名」の使い分け

`name` は ID として内部で使われ、画像ファイル名やログと連動します。**一度確定した `name` は変更しないでください。** 後から見た目を変えたい場合は `displayName` を編集してください。

## §2 既存エージェントの確認

### 2.1 UI から

画面左サイドバーの「エージェント」を開くと、登録済みエージェントがカード形式で並びます。カードには画像・表示名・説明の冒頭が見えます。

### 2.2 ファイルから

`.claude/agents/*.md` が全エージェント定義です。ファイル一覧 = エージェント一覧と考えて差し支えありません。

---

## §3 エージェントの新規追加

### 3.1 UI から追加（推奨）

1. エージェント一覧画面右上の「＋ 追加」をクリック
2. テンプレートを選ぶ（例: 空 / 秘書 / 開発者 / コビー）
3. 表示名・説明・システムプロンプトを入力
4. 「作成」で確定

確定時に `.claude/agents/{name}.md` が生成されます。

### 3.2 ファイルを直接置く

既に定義ファイルがあれば、それを `.claude/agents/` 配下に置いて KB を再読込すれば認識されます。

**注意:** `name` の重複があると後勝ちになります。追加前に既存の一覧を確認してください。

### 3.3 テンプレートの使い分け

| テンプレート | 向き |
|---|---|
| 空 | ゼロから書き起こしたい上級者向け |
| 秘書 | スケジュール管理・タスク整理に強い |
| 開発者 | コード編集・Git 操作・検証を任せる |
| コビー | KB の使い方ガイド役（既定で同梱） |

詳細は §7 を参照してください。

## §4 エージェント属性の変更

### 4.1 UI から（推奨）

エージェントのカードを開き、「編集」ボタンから以下を構造化フィールドで変更できます:

- 表示名
- 説明
- モデル
- システムプロンプト（リッチエディタ）
- 画像（§5 参照）

**属性編集は構造化フィールド経由で行ってください。** ファイルを直接編集すると、KB のバリデーションが効かず壊れる可能性があります。

### 4.2 ファイル直編集が必要なケース

- frontmatter にカスタムフィールドを追加したい（KB 未対応のため）
- 大量のエージェントをスクリプトで一括更新したい

直編集する場合は、編集後に KB を再起動してください。

### 4.3 `name`（ID）は変更しない

`name` を変更すると、紐付いている画像・セッション履歴との対応が切れます。ID を変えたい場合は「削除 → 新規作成 → 履歴の手動移行」を行ってください（v0.1.0 では履歴自動移行は未対応）。

## §5 エージェント画像の設定

### 5.1 画像の紐付けルール

エージェントには以下の 3 層で画像が紐付きます。上から順に適用されます:

1. **カスタム画像:** `.kovitoboard/assets/agents/{agent-id}/custom.{ext}` が存在すればこれを使う
2. **既定画像:** `.kovitoboard/assets/agents/{agent-id}/default.{ext}` が存在すればこれを使う
3. **フォールバック生成:** どちらも無ければ、`name` からハッシュ生成したアバターを自動表示

`{ext}` は `png` / `jpg` / `webp` のいずれかです。**ファイル名は固定**（`custom.png` 等）で、画像を差し替えたい場合は同じ名前で上書きしてください。

### 5.2 UI からのアップロード

1. エージェントのカードを開く
2. 画像部分をクリック → ファイル選択
3. 「カスタム画像として保存」 or 「既定画像として保存」を選ぶ
4. 確定すると `.kovitoboard/assets/agents/{agent-id}/` 配下に配置される

### 5.3 フォールバック生成器

画像未設定のエージェントでも、一覧カードは空にならず自動生成のアバターが表示されます。これは `name` を入力に決定的に生成されるため、同じ `name` なら常に同じアバターになります。

### 5.4 画像を削除してフォールバックに戻す

UI の「画像を削除」ボタンで、`custom.*` または `default.*` を削除できます。両方削除するとフォールバック生成に戻ります。

---

## §6 エージェントの削除

### 6.1 UI から

エージェントのカードを開き、「削除」ボタンから実行します。確認ダイアログで ID を入力する必要があります（誤操作防止）。

### 6.2 削除時の副作用

- `.claude/agents/{agent-id}.md` が削除される
- `.kovitoboard/assets/agents/{agent-id}/` 配下の画像も削除される
- **セッション履歴は保持されます**（閲覧は可能、ただし新規対話は不可）

### 6.3 元に戻したい場合

Git 管理下であれば `git checkout` で復元できます。KB 側にアンドゥ機能はありません。

## §7 推奨エージェント（v0.1.0 同梱想定）

v0.1.0 の起動時点で、以下のエージェントが最初から利用可能な状態を目指しています:

### 7.1 コビー（Kovito のコンシェルジュ）

- **ID:** `kovito-concierge`
- **役割:** KB の使い方ガイド役。このリファレンスを読みながら回答する案内役
- **ユーザーが最初に話しかける相手として推奨**

### 7.2 秘書

- **ID:** `secretary`
- **役割:** タスク整理・スケジュール管理・日報作成等
- **向いているタスク:** 毎日の「今日何する？」「今日の振り返り」

### 7.3 Kovito 開発者

- **ID:** `kovito-developer`
- **役割:** KB 自体の拡張（レシピ作成・app/ 配下の追加）支援
- **向いているタスク:** コード編集・検証・独自アプリの立ち上げ

### 7.4 カスタムエージェントを育てるコツ

- **役割を 1 つに絞る:** 「何でも屋」を作るよりも、目的特化のほうが質が上がります
- **口調を統一する:** システムプロンプト冒頭に「です/ます調」等を明記する
- **守るべきルールを列挙する:** NG 行動を書くと暴走が減ります
