# 09. データ取扱いと注意事項

**対象 KB バージョン:** v0.2.8

## 1. 概要 — KB のデータフロー

KovitoBoard は Claude Code を経由してエージェントを動かす設計です。
このため、KB の画面に表示される情報、KB に読み込まれるファイルの内容、エージェントとの対話内容は、
**Claude Code 経由で Anthropic のモデル（Claude）に渡る** ことを認識してください。

「KB に何かを読み込ませる」「KB の画面で何かを表示する」「エージェントに何かを質問する」
これらの操作は、「Claude にデータを渡す」操作と同等です。

## 2. 基本方針

KovitoBoard はフィールドレベルのマスキング、自動編集、サンドボックス化されたコンテキスト等の
データ保護機構を実装していません。

- KB のデータは Claude Code 経由で Anthropic のモデルに渡る
- Anthropic の利用規約に従ったデータ取扱いは Anthropic 側で管理
- データ学習に利用しない設定は Anthropic 側で管理（Claude Pro/Max では既定で ON）
- **機密情報を扱うアプリ・レシピでは、データ取得時のマスキング処理の実装を積極的に検討してください**

## 3. Anthropic 側の設定確認

KovitoBoard を使用する前に、Anthropic 側の以下の設定を確認することを推奨します:

- **データを学習に利用しない設定が ON か**
  - Claude Pro / Max アカウントでは既定で ON
  - Anthropic アカウント設定の "Privacy" セクションで確認可能
- **エンタープライズ契約等での追加保護**（該当する場合は契約条件を確認）

設定の詳細は Anthropic の公式ドキュメントを参照してください。

## 4. 機密情報を扱う場合の推奨対応

以下のような機密情報を扱う場合は、データ取得時のマスキング処理の実装を積極的に検討してください:

| 情報の種類 | 推奨される取扱い |
|---|---|
| パスワード | KB アプリ上での入力は想定していません。実装上も取得・表示しない設計を推奨 |
| API キー / アクセストークン | データ取得時にマスキング、KB の画面・ログに raw 値を出さない |
| 個人情報（メールアドレス / 電話 / 住所） | アプリ要件に応じてマスキングを検討（例: メールアドレスのドメイン部のみ表示） |
| 業務機密ドキュメント | Document Viewer 等で開く前にマスキング適用を検討 |
| 金融情報（残高 / 取引履歴） | 表示が必要な場合は表示単位を粗くする等の工夫 |
| 第三者著作物（配信権なき情報） | KB に読み込ませないことを推奨 |

### マスキング実装の例

ユーザー / アプリ作者がマスキング処理を実装する一般的な方法:

- **API 層での置換**: アプリの API（`app/api/*.ts` 等）でデータ取得時に機密フィールドを置換（例: `password: "..."` → `password: "[REDACTED]"`）
- **表示前のフィルタリング**: 表示直前のレンダリング層でマスキング（例: メールの `@` 以降のみ表示、API キーは末尾 4 文字のみ表示）
- **ファイル読込時の置換**: Document Viewer 等で開く前に正規表現で機密パターンを置換

KovitoBoard はマスキング実装をサポートする組込み機能を提供しませんが、レシピ / 独自アプリの実装層で
自由に組み込めます。

## 5. AmbientSidebar 固有の補足

KB v0.1.0 から導入された AmbientSidebar は、画面のコンテキストをエージェントに渡す機能を持ちます。
このとき以下の情報がエージェントに渡ります:

- **accessibility tree**: 画面の「人間に見えている構造のサマリ」（見出し、リンク、ボタン、テキスト等）
  - 自動取得、サイドバーで質問するたびに最新化
  - DOM 全体ではなく、accessibility 観点でユーザーに見える情報のみ
- **ユーザー選択範囲**: ユーザーが画面要素を選択して「これについて」と聞いた場合
- **アプリ宣言情報**: アプリ作者が `window.kb.exposeContext({...})` で公開した内部状態

これらは Claude Code 経由でモデルに渡ります。
機密情報を含む画面でサイドバーを使う場合は、`window.kb.exposeContext` で渡す情報をマスキング済みのものに
限定する等、アプリ側の対策を検討してください。

## 6. アプリ作者への推奨（独自アプリ開発時）

`app/` 配下に独自アプリを開発する場合、機密情報を扱うアプリでは以下を検討してください:

- **データ取得層でのマスキング**: API 層で機密フィールドを取得時に置換
- **`window.kb.exposeContext({...})` で渡す情報の選定**: 不要な機密情報を含めない
- **DOM への機密情報出力の最小化**: 表示しなくても済む情報は表示しない
- **データソース選定の明示**: アプリが扱うデータソースを README で明示
- **Kovito 開発者エージェント**: 独自アプリ開発時にマスキング設計を提案してくれます
